## Question
how is the login system for slimyai.xyz set-up

## Answer
The current SlimyAI login system is an email/password stack built around `lib/slimy-auth`, not Discord OAuth. Passwords are verified with Argon2, authenticated sessions are persisted via Prisma in MySQL, and session state is transported with httpOnly cookies. ([Auth and Retired Services](../wiki/architecture/auth-and-retired-services.md))

Authorization is layered on top of login: owner-only routes are protected by `requireAuth()`, so privileged panels are gated after authentication. ([Auth and Retired Services](../wiki/architecture/auth-and-retired-services.md))

Operationally, the web app that serves slimyai.xyz is a Next.js app (`slimy-web`) running on NUC2, while core infrastructure includes MySQL on NUC1; this matches the split where auth app behavior is in the web tier and session persistence is in MySQL. ([Slimy Web](../wiki/projects/slimy-web.md), [NUC Topology and Services](../wiki/architecture/nuc-topology-and-services.md))

Legacy login dependencies were intentionally removed: Discord OAuth, `admin-api` (3080), `admin-ui` (3081), and the old `/api/* -> 3080` rewrite are retired and should not be reintroduced. ([Auth and Retired Services](../wiki/architecture/auth-and-retired-services.md))
