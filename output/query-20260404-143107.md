## Question
how is the wiki setup and how was it built with the slimy-harness

## Answer
The wiki is set up as the knowledge layer of the SlimyAI harness, with a clear separation of responsibilities:
- `kb/raw/` stores source documents and session learnings.
- `kb/wiki/` stores compiled, topic-oriented knowledge articles.
- `kb/output/` stores one-off query answers like this one.
- `kb/tools/` supports KB operations (search/sync/write workflows).
This structure is explicitly described as the “Knowledge harness” in the layered harness model. [Harness Runtime Topology](../wiki/architecture/harness-runtime-topology.md)

At the content level, the wiki is organized by fixed categories (Concepts, Projects, Patterns, Troubleshooting, Architecture) and indexed centrally in `_index.md`, with `_concepts.md` as a flat concept registry. The index is marked as auto-maintained by LLM agents. [_index](../wiki/_index.md), [_concepts](../wiki/_concepts.md)

How it was built with slimy-harness (from KB evidence):
1. The global harness contract defines strict startup/session discipline (read ledgers and session context first, then execute under repo rules). [Agent Session Contract](../wiki/concepts/agent-session-contract.md), [Source of Truth Ledgers](../wiki/concepts/source-of-truth-ledgers.md)
2. During/after sessions, learnings are captured and promoted from short-term logs into durable operating docs, which is the pattern used to create durable KB content from execution history. [Memory Capture Pattern](../wiki/patterns/memory-capture-pattern.md)
3. Closeout rules require verification and ledger updates, which provides the evidence trail that feeds reliable documentation updates over time. [Session Closeout Pattern](../wiki/patterns/session-closeout-pattern.md), [Truth Gate](../wiki/concepts/truth-gate.md)
4. The KB is treated as shared infrastructure in the harness topology (host-local files, shared via repo sync channels across NUCs), so it is built incrementally by repeated agent sessions rather than one-time manual authoring. [Harness Runtime Topology](../wiki/architecture/harness-runtime-topology.md), [Cross-NUC Communication Matrix](../wiki/architecture/cross-nuc-communication-matrix.md)

In short: the wiki is the compiled documentation layer of slimy-harness, built continuously from session artifacts and governed by startup/closeout/truth-gate rules.
