# KB Page Type Conventions

> This doc describes the 9 standard wiki page types, their required sections, filename patterns, and backlink/ Sources handling.
> Legacy pages using `> Category:` frontmatter remain valid. New and updated pages should use YAML frontmatter with `page_type`.

## The 9 Page Types

### 1. entity
**What:** A named thing with a stable identity — person, service, repo, bot, host.
**Filename:** `kebab-case.md` matching the entity name.
**Required sections:** Description, Runtime State (if applicable).
**Optional:** entity_type key (service, person, bot, repo, host).
**Sources:** Raw decision docs, inventory docs.
**Backlinks:** Link to related services/processes; link from overview/summary pages.

### 2. concept
**What:** An abstract idea, pattern, or principle that can be reused across many pages.
**Filename:** `kebab-case.md` in `concepts/` or `patterns/` category.
**Required sections:** Definition, Why It Matters.
**Optional:** Related concepts listed at bottom.
**Sources:** Agent learnings, research docs.
**Backlinks:** Referenced by all pages that use this concept; listed in _concepts.md.

### 3. source
**What:** A raw input document — decision record, research finding, changelog, idea.
**Filename:** Reflects source path (e.g., `YYYY-MM-DD-slug.md`).
**Required sections:** Summary, Sources (raw provenance).
**Optional keys:** source_path, source_kind, source_date.
**Sources:** Should point to original raw/ path.
**Backlinks:** Compiled-from pages link back; index pages list sources by kind.

### 4. comparison
**What:** Side-by-side analysis of two or more options.
**Filename:** `thing-a-vs-thing-b.md` or `thing-comparison.md`.
**Required sections:** Options, Comparison Matrix, Recommendation.
**Optional keys:** compared_items (array).
**Sources:** Research docs, decision docs.
**Backlinks:** Each option's entity page should link to the comparison.

### 5. decision
**What:** A recorded decision with rationale and status.
**Filename:** `YYYY-MM-DD-decision-slug.md`.
**Required sections:** Decision, Rationale, Status.
**Optional keys:** decision_date, decision_status (accepted/rejected/deprecated).
**Sources:** Meeting notes, RFC docs, experiment results.
**Backlinks:** Referenced by entity pages affected by the decision.

### 6. overview
**What:** A high-level summary or state-of-the-world page (machine, project, system).
**Filename:** `overview.md`, `<project>-overview.md`, or `<host>-state.md`.
**Required sections:** Summary, Components, Current State.
**Sources:** Inventory docs, status docs, runtime docs.
**Backlinks:** Components link back to overview; _index entry is required.

### 7. troubleshooting
**What:** A problem→solution record with prevention.
**Filename:** `problem-description.md`.
**Required sections:** Symptom, Cause, Fix, Prevention.
**Sources:** Incident notes, debug sessions.
**Backlinks:** Entity pages for affected components link here.

### 8. index
**What:** A curated list of related pages (e.g., _index.md, _concepts.md).
**Filename:** `_name.md` (underscore prefix).
**Required sections:** None (list format).
**Special:** Index pages are NOT counted as orphans (they have no incoming links requirement).
**Sources:** Auto-generated by compile; manually maintained for quality.

### 9. log
**What:** An append-only event or activity log.
**Filename:** `log.md` or `YYYY-MM-DD-activity-log.md`.
**Required sections:** None (chronological entries).
**Special:** Log entries never edited after creation.
**Sources:** System events, maintenance runs, agent sessions.

## YAML Frontmatter Schema

All types support this base frontmatter:

```yaml
---
title: "<page title>"
page_type: "<entity|concept|source|comparison|decision|overview|troubleshooting|index|log>"
status: "active"
tags: []
updated: YYYY-MM-DD
source_count: 0
aliases: []
---
```

**Type-specific optional keys:**
- source: `source_path`, `source_kind`, `source_date`
- comparison: `compared_items: [...]`
- decision: `decision_date: YYYY-MM-DD`, `decision_status: accepted|rejected|deprecated`
- entity: `entity_type: service|person|bot|repo|host`

## Backlink Conventions

- Use relative paths: `[see also](../patterns/foo.md)`
- Every non-index page should be reachable from at least one other non-index page
- Orphaned pages (zero inbound links from other non-index pages) are tracked in `_orphans.md`
- Low-connectivity pages tracked in `_weak-links.md`

## Sources Field

Wiki pages compiled from raw docs use the `> Sources:` line at the top:
```
> Sources: raw/decisions/2026-04-09-project-chat-app.md, raw/research/2026-04-09-nuc2-project-inventory.md
```

The frontmatter `source_count` tracks how many raw sources are referenced.
